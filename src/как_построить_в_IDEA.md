# Как построить проект в IDEA JETBRAINS

**IntelliJ IDEA (IDEA)** — это самая популярная и мощная IDE для Java, она значительно упрощает процесс сборки по сравнению с командной строкой.

В IntelliJ IDEA термин "построить проект" (Build Project) означает компиляцию всех исходных файлов (`.java`) в байт-код (`.class`) и размещение их в выходном каталоге.

Вот пошаговое руководство, как это сделать в IDEA, с учетом современных подходов (через системы сборки).

-----

### Часть 1: Сборка простого модуля (Без Maven/Gradle)

Если вы создали простой Java-проект, который IDEA называет **Module**, вы можете использовать встроенную систему сборки IDE.

1.  **Откройте проект** в IntelliJ IDEA.
2.  **Проверьте конфигурацию:**
    * Перейдите в `File` $\rightarrow$ `Project Structure...` (или нажмите `Ctrl+Alt+Shift+S` / `⌘+;`).
    * Убедитесь, что для вашего модуля правильно указан **SDK** (Java Development Kit, например, JDK 17).
3.  **Запустите сборку:**
    * В главном меню перейдите в `Build` $\rightarrow$ **`Build Project`** (или нажмите сочетание клавиш `Ctrl+F9` / `⌘+F9`).
    * IDEA скомпилирует все `.java` файлы в вашем проекте.
4.  **Результат:**
    * Скомпилированные `.class` файлы будут помещены в выходной каталог, который обычно называется `out/production/НазваниеМодуля`.
    * В нижней части экрана в окне **"Build"** появится сообщение о том, что сборка завершена успешно (`Build completed successfully`).

-----

### Часть 2: Сборка проекта с системой управления зависимостями (Maven/Gradle)

В современных Java-проектах почти всегда используются системы сборки (такие как **Maven** или **Gradle**), которые управляют зависимостями, автоматизируют компиляцию и создают исполняемые архивы (JAR/WAR).

**IntelliJ IDEA имеет встроенную поддержку для этих систем.**

#### A. Сборка через панель инструментов IDEA (Рекомендуемый способ)

IDEA автоматически импортирует Maven (файл `pom.xml`) или Gradle (файл `build.gradle`) и создает на их основе свою структуру проекта.

1.  **Откройте панель инструментов (Tool Window):**
    * Справа в окне IDEA найдите вкладки **`Maven`** или **`Gradle`** и нажмите на них.
2.  **Запуск жизненного цикла сборки:**
    * В открывшемся окне (Maven Projects или Gradle):
        * Разверните ваш проект.
        * Разверните папку **`Lifecycle`**.
        * Для стандартной сборки дважды кликните на цель **`install`** (Maven) или **`build`** (Gradle).
3.  **Результат:**
    * Система сборки выполнит компиляцию, запустит тесты, и, самое главное, создаст **JAR-файл** вашего приложения.
    * Этот исполняемый файл будет находиться в каталоге `target/` (для Maven) или `build/libs/` (для Gradle).

#### B. Сборка через терминал IDEA

Вы также можете использовать терминал, встроенный в IDEA, для запуска команд Maven или Gradle напрямую:

1.  **Откройте терминал:** Нажмите `Alt+F12` или найдите вкладку **`Terminal`** внизу окна IDEA.
2.  **Запустите команду сборки:**
    * **Для Maven:**
      ```bash
      ./mvnw clean install 
      # или 
      mvn clean install 
      ```
    * **Для Gradle:**
      ```bash
      ./gradlew build 
      # или 
      gradle build
      ```
    * Команда `clean` очищает предыдущие результаты сборки, а `install` или `build` выполняет всю цепочку (компиляция, тестирование, упаковка).

### Часть 3: Создание исполняемого JAR-файла (Artifact)

Если вы хотите создать один исполняемый файл `.jar`, 
который можно запустить из командной строки командой `java -jar myapp.jar`, вам нужно настроить "Артефакт" (Artifact).

1.  **Перейдите в `File` $\rightarrow$ `Project Structure...`**
2.  В левом меню выберите **`Artifacts`**.
3.  8Нажмите на знак **плюса (`+`)** и выберите:
    * **JAR** $\rightarrow$ **From modules with dependencies...**
4.  **Настройте Артефакт:**
    * **Main Class:** Укажите ваш класс, содержащий метод `public static void main(String[] args)`.
    * IDEA автоматически соберет все необходимые зависимости.
5.  **Создайте Артефакт:**
    * Перейдите в меню `Build` $\rightarrow$ **`Build Artifacts...`**
    * Выберите созданный вами артефакт и нажмите **`Build`**.
    * Исполняемый JAR-файл появится в каталоге `out/artifacts/НазваниеАртефакта`.
6.  **Запустите Артефакт:**
    * Перейдите в Терминал, перейдите в каталог `out/artifacts/НазваниеАртефакта` и запустите файл артефакта `java -jar myapp.jar`

-----

**Резюме:**

| Цель | Способ в IDEA | Горячая клавиша |
| :--- | :--- | :--- |
| **Просто скомпилировать** `.java` в `.class` | `Build` $\rightarrow$ `Build Project` | `Ctrl+F9` / `⌘+F9` |
| **Полная сборка** (с тестами, упаковкой JAR) | `Maven` / `Gradle` $\rightarrow$ `install`/`build` | Нет прямой |
| **Создать исполняемый JAR-файл** | `Build` $\rightarrow$ `Build Artifacts...` | Нет прямой |
