 ## Запуск Java-проекта из командной строки

Запуск Java-проекта из командной строки — это базовый навык для любого Java-разработчика.  
Процесс состоит из двух основных шагов: компиляция исходного кода и запуск скомпилированного кода.

Вот пошаговое руководство, как это сделать для простого проекта.

### Шаг 1: Подготовка (Написание кода)

Предположим, у вас есть простой файл с исходным кодом Java.

Создайте файл с именем HelloWorld.java:

```Java

// HelloWorld.java

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Привет, мир! Я запущен из командной строки.");
    }
}
```

### Шаг 2: Компиляция (Создание байт-кода)

Чтобы запустить этот код, его сначала нужно скомпилировать в байт-код, который может выполнить виртуальная машина Java (
JVM).

Откройте командную строку (или терминал).

Перейдите в каталог, где находится ваш файл HelloWorld.java.

Например, если ваш файл находится в /Users/UserName/JavaProjects/MyProject, используйте:

```Bash

cd /Users/UserName/JavaProjects/MyProject
```

Скомпилируйте файл, используя компилятор javac:

```Bash

javac HelloWorld.java
```

Результат: Если компиляция прошла успешно, в том же каталоге появится новый файл с именем HelloWorld.class. Это и есть
скомпилированный байт-код.

### Шаг 3: Запуск (Выполнение байт-кода)

Теперь вы можете запустить скомпилированный класс, используя исполняющую среду Java (java).

Запустите программу, используя команду java, указав только имя класса (без расширения .class):

```Bash

java HelloWorld
```

Результат: Ваша программа выполнится, и вы увидите вывод в консоли:

```Привет, мир! Я запущен из командной строки.```

### Продвинутый подход: Работа с несколькими файлами и пакетами (Packages)

В реальных проектах код обычно организован в пакеты (packages) и состоит из множества файлов.

1. Структура каталогов с пакетами
   Если у вас есть код в пакете, например com.example.app, структура каталогов должна это отражать:
```
MyProject/
├── src/
│ └── com/
│ └── example/
│ └── app/
│ └── MainApp.java
└── bin/  <-- Сюда будут помещаться скомпилированные .class файлы
```

Содержимое MainApp.java:


```Java

// src/com/example/app/MainApp.java

package com.example.app;

public class MainApp {
    public static void main(String[] args) {
        System.out.println("Запуск из пакета!");
    }
}
```

2. Компиляция с пакетами
   Вам нужно указать javac, где находится исходный код (-sourcepath или -d) и куда поместить скомпилированные файлы (
   -d).

Убедитесь, что вы находитесь в корневом каталоге проекта (MyProject).

Создайте каталог bin, если его нет: mkdir bin

Компиляция:

```Bash

javac -d bin src/com/example/app/MainApp.java
```

Ключ -d bin говорит компилятору: "Помести скомпилированные .class файлы в каталог bin, сохраняя при этом структуру
пакетов."

3. Запуск с пакетами
   Для запуска необходимо указать, где находится скомпилированный код, используя classpath (-cp), и затем указать полное
   имя класса, включая пакет.

Запуск:

```Bash

java -cp bin com.example.app.MainApp
```

Ключ -cp bin говорит JVM: "Ищи классы для запуска в каталоге bin."

com.example.app.MainApp — это полное имя вашего основного класса.

### Рекомендация для реальных проектов

Хотя знать, как использовать javac и java напрямую, полезно, в реальных, сложных проектах почти всегда используются
системы сборки (Build Systems), такие как Maven или Gradle.

Эти инструменты автоматизируют:

Компиляцию всех файлов.

Управление зависимостями (внешними библиотеками).

Создание исполняемых JAR-файлов.

Например, для запуска проекта, созданного с помощью Maven, обычно достаточно выполнить команду:

```Bash

mvn compile exec:java
```
